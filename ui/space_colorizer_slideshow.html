Architecture

The U-Net

⬇ Encoder (Contracting Path)
DoubleConv blocks (2× Conv2d + BatchNorm + ReLU) → MaxPool2d. Each level doubles channels: 1 → 64 → 128 → 256 → 512. Captures "what" is in the image.
Skip Connections
Feature maps from each encoder level are concatenated to the corresponding decoder level, preserving spatial detail (edges, textures) lost during downsampling.
⬆ Decoder (Expanding Path)
ConvTranspose2d upsamples, halving channels each level. Final 1×1 Conv outputs 3 channels (RGB). Answers "where" colors belong.
class DoubleConv(nn.Module): # Core building block def __init__(self, in_ch, out_ch): super().__init__() self.net = nn.Sequential( nn.Conv2d(in_ch, out_ch, 3, padding=1), nn.BatchNorm2d(out_ch), nn.ReLU(inplace=True), nn.Conv2d(out_ch, out_ch, 3, padding=1), nn.BatchNorm2d(out_ch), nn.ReLU(inplace=True) ) # Input: 1 ch grayscale # Output: 3 ch RGB model = UNet(in_ch=1, out_ch=3)

Training Signal

Loss Functions

📐
L1 Loss
Mean Absolute Error. Penalizes large color deviations linearly. Robust to outliers. Weight: 1.0×
📏
MSE Loss
Mean Squared Error. Squares the error, heavily penalizing large deviations. Weight: 0.5×
🎨
Perceptual Loss
VGG16 feature comparison, forcing texture & structure matching rather than just pixel averages. Weight: 0.01×
combined_loss = L1(pred, target) + 0.5 × MSE(pred, target) + 0.01 × VGG(pred, target)
The 0.01 VGG weight prevents the perceptual loss from overpowering pixel-level accuracy, tuned to add color vibrancy without distorting shapes.

Key Challenge

The Muted Color Problem

Why It Happens
L1/MSE loss pushes the model toward the statistical mean color of the training set. For Hubble images, warm dust tones dominate, so the model "plays it safe" with brownish-amber for everything.
Classic ML Problem Name
Called "regression to the mean" or the "color desaturation" problem that affects all pixel-wise loss colorization models. Even DeOldify and similar models struggled with this.
Fix #1: Perceptual Loss (VGG16)
Added VGG16 feature comparison. Forces the model to match textures and structure, not just pixel averages. Noticeably more vivid colors by epoch 35–40.
Fix #2: Post-Processing
Color.enhance(2.0) + Contrast.enhance(1.2) + Sharpness.enhance(1.3). Amplifies whatever the model predicted honestly, with no color remapping.

The Product

Gradio Web App

🖼️ Input
Accepts JPG/PNG grayscale telescope images. User drags & drops with no resizing required. Model handles 256×256 internally.
⚡ Inference Pipeline

Honest Assessment

Results & Limitations

✓ What Works
✓ Carina Nebula pillars show convincing warm amber/rust tones; the model learned real color-structure relationships
✓ No overfitting: train/val loss converged tightly at epoch 30 (0.035 vs 0.034)
✓ Ring Nebula gold/orange tones markedly improved from epoch 8 → 30
⚠ Known Limitations
⚠ Output not scientifically accurate; predicts plausible colors, not true astrophysical emission lines
⚠ Dataset of 1,612 images is small; more data would significantly improve generalization on novel objects
⚠ Background space defaults to amber instead of cool blue-teal; the model saw more warm nebula than cold space background in training

What's Next

Future Directions

Scale the Dataset
Target 5,000–10,000 Hubble/JWST images. Script bulk downloads from NASA MAST archive. More data is the single biggest quality lever.
GAN Architecture
Replace pixel loss with adversarial training (Pix2Pix or conditional GAN). Discriminator learns "what does a real Hubble image look like," eliminating color averaging.
Transformer Backbone
Replace the CNN encoder with a Swin Transformer or ViT. Transformers capture global context, better suited for handling entire nebulae as coherent scenes.
